## 2. STRATÉGIE GÉNÉRALE DE LA CAMPAGNE

### Architecture des séquences
- **3 séquences parallèles** basées sur 3 douleurs différentes
- **Chaque séquence** : 3 emails sur 10 jours (J0, J+3, J+7)
- **Approche multi-canal** : Une séquence par problème de conformité
- **Segmentation intelligente** : Sélection selon signaux de non-conformité visible

### Workflow par prospect
```
Prospect identifié (LinkedIn/Infogreffe)
    ↓
Audit de conformité visible (site web, mentions RGPD, etc.)
    ↓
Attribution à séquence (A, B ou C)
    ↓
Email 1 (J0) - Hook / Audit gratuit
    ↓
Pause 3 jours + tracking
    ↓
Email 2 (J+3) - Preuve sociale / Case study
    ↓

    ↓

```

## 7. CONFIGURATION BREVO SMTP

### Caractéristiques compte Brevo Free Tier
```
Limite quotidienne : 300 emails/jour
Limite mensuels : 9 000 emails/mois
Réputation IP : Partagée (plus prudent au démarrage)
SPF/DKIM/DMARC : Support natif
Délai de chauffe ("warm-up") : Recommandé sur 2 semaines
```

### Plan d'échauffement (warm-up) 2 semaines
```
Semaine 1 :
  J1-J2 : 50 emails/jour (total 100)
  J3-J4 : 75 emails/jour (total 150)
  J5-J6 : 100 emails/jour (total 200)
  J7   : 100 emails/jour (total 100)

Semaine 2 :
  J8-J10 : 150 emails/jour (total 450)
  J11-J12: 200 emails/jour (total 400)
  J13-J14: 200 emails/jour (total 400)

Semaine 3+ : Montée progressive jusqu'à 300/jour max
```

**Raison** : Éviter que Gmail/Outlook marque le domaine comme spammeur dès le départ.

### Configuration SPF/DKIM/DMARC

#### 1. SPF (Sender Policy Framework)
Ajouter dans votre DNS zone (par ex. nexusconformite.fr) :

```dns
nexusconformite.fr. IN TXT "v=spf1 include:sendingservice.brevo.com ~all"
```

Vérifier propagation (2-24h) :
```bash
nslookup -q=txt nexusconformite.fr
dig nexusconformite.fr TXT
```

#### 2. DKIM (DomainKeys Identified Mail)
Brevo fournit une clé DKIM personnalisée. Enregistrement type :

```dns
brevo._domainkey.nexusconformite.fr. IN TXT "v=DKIM1; k=rsa; p=MIGfMA0GCS..."
```

Procédure Brevo :
1. Login Brevo → Settings → Sending domain
2. Valider domaine
3. Copier clés DKIM fournis
4. Ajouter dans DNS zone
5. Attendre propagation + validation Brevo

#### 3. DMARC (Domain-based Message Authentication, Reporting & Conformance)
Enregistrement DMARC recommandé :

```dns
_dmarc.nexusconformite.fr. IN TXT "v=DMARC1; p=quarantine; rua=mailto:dmarc@nexusconformite.fr; ruf=mailto:dmarc@nexusconformite.fr; fo=1"
```

**Explication** :
- `p=quarantine` : Messages non-signés → SPAM (prudent)
- `rua=` : Rapports agrégés (quotidiens)
- `ruf=` : Rapports forensic (pour investigation)
- `fo=1` : Rapport si SPF OU DKIM échoue

### Gestion des rebonds dans Brevo

**Types de rebonds**
- **Hard bounce** (erreur permanente) : Domaine inexistant, compte supprimé
  - Action : Suppression automatique après 1 rebond
  - Impact : Ne pas relancer ces adresses

- **Soft bounce** (erreur temporaire) : Boîte pleine, serveur indisponible
  - Action : Retry automatique 3x (3j, 6j, 10j)
  - Après 3 soft : Marqué comme hard et supprimé

**Configuration dans Brevo**
1. Automation → Bounce handling
2. Hard bounce threshold : Automatique (default)
3. Soft bounce limit : 3 tentatives (default)
4. Blacklist auto-update : Activé

**À surveiller** :
- Taux de bounce < 2% (excellent < 1%)
- Si > 3%, revoir liste prospect (qualité Infogreffe/LinkedIn)

## 11. MATRICE DE DÉCISION : QUI REÇOIT QUELLE SÉQUENCE ?

### Critères de segmentation

```
SI (site web EXISTS) AND (aucune mention RGPD visible)
   THEN → Séquence A (RGPD Urgency)
        Indicateurs : Pas de "politique confidentialité",
                      pas de banneau cookies,
                      formulaire non-compliant

SI (secteur = Distribution/Services/Tech)
   AND (pas évident NF525 or Axonaut/PayFit/Pennylane)
   THEN → Séquence B (Facturation 2026)
        Indicateurs : Logiciel ancien (Ciel, Sage),
                      pas mention NF525,
                      site rétrospect (design 2020-)

SI (effectifs visible >50 salariés)
   AND (domaine technologie faible visible ou ancien site)
   THEN → Séquence C (NIS2 Cybersécurité)
        Indicateurs : Pas RSSI/CSO,
                      peu contrôles accès visibles,
                      email server basique,
                      retard patch visible (e.g., site vuln)

BONUS: SI (effectif 50-150 AND secteur Tech/Services)
       THEN → Envoi DOUBLE (Seq A + Seq C, 4-5j décalage)
            Raison : PME très exposées RGPD + NIS2
            Limit : 1-2 contacts max per company
```
